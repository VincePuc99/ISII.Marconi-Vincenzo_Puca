16-Corso sugli script bash - aggiunta di una GUI

La GUI è l'acronimo inglese per Graphical User Interface, ovvero di interfaccia grafica. Ebbene sì, dopo aver lavorato per 15 lezioni del nostro corso con il terminale, possiamo ora fare un passo in più, anche se non obbligatorio, e aggiungere un'interfaccia grafica allo script. Per gli ambienti grafici basati sulle librerie GTK, come Gnome e quindi Ubuntu, esiste un modulo chiamato zenity che permette di integrare comodamente e facilmente delle finestre grafiche sempre mediante comandi.
Il pacchetto zenity dovrebbe già essere installato per difetto in Ubuntu, ma se non fosse il caso basta aprire Ubuntu Software Center e cercare "zenity".

Opzioni disponibili con il comando zenity:

Miscellaneous options

  • -?, --help → Show summary of options (aiuto sulle diverse opzioni)
  • --about → Display an about dialog (dialogo con info su zenity e versione)
  • --version → Show version of program (numero di versione nel terminale)

General options

  • --title=TITLE → Set the dialog title
  • --window-icon=ICONPATH → Set the window icon with the path to an image. Alternatively, one of the four stock icons can be used: 'error', 'info', 'question' or 'warning'
  • --width=WIDTH → Set the dialog width
  • --height=HEIGHT → Set the dialog height
  • --timeout=TIMEOUT → Set the dialog timeout in seconds

Dialog options

  • --calendar → Display calendar dialog
    • Calendar options
      • --text=STRING → Set the dialog text
      • --day=INT → Set the calendar day
      • --month=INT → Set the calendar month
      • --year=INT → Set the calendar year
      • --date-format=PATTERN → Set the format for the returned date. The default depends on the user locale or be set with the strftime style. For example %A %d/%m/%y (rif. http://php.net/manual/en/function.strftime.php)
  • --entry → Display text entry dialog
    • Text entry options
      • --text=STRING → Set the dialog text
      • --entry-text=STRING → Set the entry text
      • --hide-text → Hide the entry text (nasconde quanto si digita in stile password)
  • --error → Display error dialog
    • Error options
      • --text=STRING → Set the dialog text
      • --no-wrap → Do not enable text wrapping
      • --no-markup → Do not enable pango markup (rif. http://developer.gnome.org/pygtk/2.22/pango-markup-language.html)
  • --file-selection → Display file selection dialog
    • File selection options
      • --filename=FILENAME → Set the file or directory to be selected by default
      • --multiple → Allow selection of multiple filenames in file selection dialog
      • --directory → Activate directory-only selection
      • --save → Activate save mode
      • --separator=SEPARATOR → Specify separator character when returning multiple filenames
      • --confirm-overwrite → Confirm file selection if filename already exists
      • --file-filter=NAME | PATTERN1 PATTERN2 → Sets a filename filter
  • --info → Display info dialog
    • Info options
      • --text=STRING → Set the dialog text
      • --no-wrap → Do not enable text wrapping
      • --no-markup → Do not enable pango markup (rif. http://developer.gnome.org/pygtk/2.22/pango-markup-language.html)
  • --list → Display list dialog
    • List options
      • --text=STRING → Set the dialog text
      • --column=STRING → Set the column header
      • --checklist → Use check boxes for first column
      • --radiolist → Use radio buttons for first column
      • --separator=STRING → Set output separator character
      • --multiple → Allow multiple rows to be selected
      • --editable → Allow changes to text
      • --print-column=NUMBER → Specify what column to print to standard output. The default is to return the first column. 'ALL' may be used to print all columns.
      • --hide-column=NUMBER → Hide a specific column
      • --hide-header → Hides the column headers
  • --notification → Display notification
    • Notification options
      • --text=STRING → Set the notification text
      • --listen → Listen for commands on stdin. Commands include 'message', 'tooltip', 'icon', and 'visible' separated by a colon.
For example, 'message: Hello world', 'visible: false', or 'icon: /path/to/icon'. The icon command also accepts the four stock icon: 'error', 'info', 'question', and 'warning'
  • --progress → Display progress indication dialog
    • Progress options
      • --text=STRING → Set the dialog text
      • --percentage=INT → Set initial percentage
      • --auto-close → Close dialog when 100% has been reached
      • --auto-kill → Kill parent process if cancel button is pressed
      • --pulsate → Pulsate progress bar
      • --no-cancel → Hides the cancel butto
  • --question → Display question dialog
    • Question options
      • --text=STRING → Set the dialog text
      • --no-wrap → Do not enable text wrapping
      • --no-markup → Do not enable pango markup
      • --ok-label=TEXT → Set the text of the OK button
      • --cancel-label=TEXT → Set the text of the cancel button
  • --text-info → Display text information dialog
    • Text options
      • --filename=FILENAME → Open file
      • --editable → Allow changes to text
      • --checkbox=TEXT → Enable a checkbox for use like a 'I read and accept the terms.'
      • --ok-label=TEXT → Set the text of the OK button
      • --cancel-label=TEXT → Set the text of the cancel button
  • --warning → Display warning dialog
    • Warning options
      • --text=STRING → Set the dialog text
      • --no-wrap → Do not enable text wrapping
      • --no-markup → Do not enable pango markup (rif. http://developer.gnome.org/pygtk/2.22/pango-markup-language.html)
  • --scale → Display scale dialog
    • Scale options
      • --text=STRING → Set the dialog text
      • --value=VALUE → Set initial value
      • --min-value=VALUE → Set minimum value
      • --max-value=VALUE → Set maximum value
      • --step=VALUE → Set step size
      • --print-partial → Print partial values (visualizza nel terminale ogni valore mentre si sposta il cursore sulla scala)
      • --hide-value → Hide value
  • --color-selection → Display color selection dialog
    • Color selection options
      • --color=VALUE → Set the initial color (un valore esadecimale come #123456, oppure il nome inglese del colore tipo "orange", "white")
      • --show-palette → Show the palette (mostra una serie aggiuntiva di colori predefiniti da scegliere)
  • --password → Display password dialog
    • Password dialog options
      • --username → Display the username field
  • --forms → Display forms dialog
    • Forms dialog options
      • --add-entry=FIELDNAME → Add a new Entry in forms dialog
      • --add-password=FIELDNAME → Add a new Password Entry in forms dialog
      • --add-calendar=FIELDNAME → Add a new Calendar in forms dialog
      • --text=STRING → Set the dialog text
      • --separator=STRING → Set output separator character
      • --forms-date-format=PATTERN → Set the format for the returned date. The default depends on the user locale or be set with the strftime style. For example %A %d/%m/%y

Interazione con i dialoghi di zenity

Naturalmente non è sufficiente essere in grado di impostare un dialogo con zenity, ma occorre anche interagire con esso in modo da intepretare nel nostro script quanto ogni dialogo ritorna dopo essere stato chiuso. Ci sono dialoghi che tornano un solo valore, per esempio quale pulsante è stato premuto, così come dialoghi che tornano più valori in una stringa, per esempio il dialogo forms che torna tanti valori quanti campi abbiamo creato. Per entrambe le situazioni ci sono delle modalità da usare nel codice del nostro script che rendono tutto sommato facile il compito.
