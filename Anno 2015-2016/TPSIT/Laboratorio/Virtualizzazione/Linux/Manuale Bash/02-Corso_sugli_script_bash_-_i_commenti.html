02-Corso sugli script bash - i commenti

Commentare il codice è importante

  • per noi stessi se a distanza di tempo vogliamo capire facilmente cosa abbiamo fatto. Infatti non è per niente scontato che ciò che è chiaro ora a mente fresca mentre lo facciamo, lo sia ancora altrettanto a distanza di tempo.
  • per gli altri se abbiamo intenzione di distribuire il nostro codice. Un vecchio adagio dice: 100 programmatori, 100 soluzioni diverse. Quindi è molto importante commentare il proprio codice in modo da facilitarne la comprensione ad altri che non necessariamente vedono e risolvono le cose come noi.
  • per le istruzioni d'uso. Dove meglio indicare come usare uno script se non nel commento iniziale? Aprendo lo script è quello ciò che vediamo per primo
  • per segnalare la licenza. Se intendiamo distribuire il nostro lavoro con tutte le caratteristiche del software open source, è buona cosa mettere nel commento iniziale sotto quale licenza è rilasciato.

  • la prima riga di uno script "#!/bin/bash" è in forma di commento, ma è "speciale", nel senso che indica al sistema quale shell, fra le varie esistenti, deve essere usata per eseguire lo script. È importante specificare la shell perché non tutte supportano lo stesso quantitativo di comandi
  • il carattere "#" si usa per commentare una singola riga
  • per commentare un blocco di più righe usare la forma seguente ⇒ :<<commento (blocco di righe) commento, dove l'etichetta "commento" permette di aprire e chiudere il blocco. Per blocchi di commento diversi, dare nomi diversi (per es. :<<comm1 (...) comm1 :<<comm2 (...) comm2, ecc.) ⇒ questa modalità diventa molto utile se vogliamo concentrarci su una parte dello script evitando di doverlo sempre eseguire per intero.
