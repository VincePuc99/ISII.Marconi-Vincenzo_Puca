11-Corso sugli script bash - le funzioni

Nel caso in cui il programma, o script nel nostro caso:

  • diventa particolarmente complesso
  • deve ripetere alcune operazioni a determinate condizioni
  • ha una parte di codice che può venirci utile in più occasioni

allora è utile suddividerlo in parti distinte mediante le funzioni, così:

  • lo rendiamo più flessibile
  • lo rendiamo più leggibile
  • lo rendiamo più razionale evitando di replicare più volte lo stesso codice
  • sarà più facile la sua manutenzione dovendo modificare solo il codice di una funzione e non le diverse repliche dello stesso codice sparse nello script

Le funzioni sono un costrutto che permette di richiamare un pezzo di codice da altre parti dello script, e quindi di avere un solo esemplare di questo codice.
Le funzioni possono avere o meno dei parametri. La dichiarazione delle funzioni deve essere fatta prima del loro uso perché uno script bash è eseguito dall'inizio verso la fine sequenzialmente.
Le funzioni possono essere ricorsive, ovvero all'interno del loro codice possono richiamare se stesse. Può essere utile se si vuole forzare l'utente ad impostare un determinato valore prima di proseguire → finché non è dato quel valore la funzione può continuare a richiamare se stessa.

  • nome_funzione() { .... }
