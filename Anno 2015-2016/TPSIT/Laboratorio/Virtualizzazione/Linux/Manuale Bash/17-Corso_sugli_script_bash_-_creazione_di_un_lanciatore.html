17-Corso sugli script bash - creazione di un lanciatore

Concludiamo il nostro percorso con un'ultima operazione che contribuisce ulteriormente ad integrare lo script nel nostro sistema Ubuntu come se fosse un vero e proprio programma. Creiamo un file lanciatore con estensione .desktop che ci permetterà di trovarlo nella Dash assieme a tutti gli altri programmi.

  • un modello di lanciatore da seguire l'ho preparato e l'avete a disposizione nell'archivio che potete scaricare dal commento al primo video di questo corso
    • modificare il lanciatore con il flag di esecuzione disattivato o attivato è la stessa cosa. Per aprirlo trascinarlo in gedit già aperto e non con il doppio clic
    • proprietà Version: se si desidera si può indicare una versione dello script
    • proprietà Name: dare il nome del programma, che apparirà sia nel file manager, sia nella Dash. Name[it_CH] è la versione della lingua specifica del vostro sistema
    • proprietà Terminal: impostare "true" se lo script deve essere eseguito aprendo un terminale; impostare "false" se esiste un'interfaccia grafica e un terminale non è necessario
    • proprietà Exec: il percorso dello script bash da eseguire. Non è necessario anteporre il comando "bash" se anche lo script ha il flag di esecuzione attivato
    • proprietà Comment e GenericName: sono rispettivamente il commento e la descrizione nelle proprietà del file tipo .desktop
    • proprietà Icon: mettere il percorso ad un file immagine che fungerà da icona per il nostro script
  • il lanciatore infine è da archiviare in ~/.local/share/applications così verrà preso automaticamente in considerazione dai meccanismi di ricerca della Dash

Siamo così giunto alla fine di questo corso :-)
