13-Corso sugli script bash - parametri di input e codice di ritorno

Adesso che abbiamo imparato a costruire il corpo del nostro script con tutta la logica necessaria, vediamo come possiamo comunicare allo script delle informazioni in entrata, e come possiamo leggere il codice di ritorno all'uscita dallo script.
I parametri in entrata possono essere più di uno, mentre il codice di ritorno è uno solo.

I parametri in entrata vengono impostati dopo il nome dello script nella sequenza prevista e all'interno dello script vengono letti con il simbolo dollaro e un numero progressivo. Per esempio se abbiamo uno script che si aspetta 3 parametri lo chiameremo così:
nicola@pangolino-pb:~$ bash script.sh val1 val2 val3
e dentro lo script i tre valori sono disponibili nelle variabili $1, $2, $3

Il codice di ritorno è dato dal comando "exit" se esplicitamente inserito nello script. Se è senza parametri ritorna il codice di ritorno dell'ultimo comando eseguito nello script, altrimenti possiamo fornire noi un codice a piacimento per indicare lo stato d'uscita dello script. Per esempio "exit 10".
Se tutto funziona correttamente il codice di ritorno è zero, se invece ci sono errori il codice è diverso da zero. Per testare il codice di ritorno di uno script, dopo averlo eseguito, usare il comando "echo $?", oppure se lo script è stato richiamato da un altro script usare la variabile $? per verificare se è andato tutto bene.

⇒ per difetto non è necessario includere il comando exit nello script perché comunque il codice ritorno dell'ultimo comando viene impostato all'uscita dallo script. Lo si può usare se vogliamo gestire noi stessi dei codici errore personalizzati per il nostro script.
